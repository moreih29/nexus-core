# Migration Guide: v0.9.x → v0.10.0

**Release date**: 2026-04-16
**Type**: Minor release (additive-with-obligation — body_hash 변경 + vocabulary/schema 확장)
**Affected consumers**: claude-nexus, opencode-nexus
**Tracking issues**: [GH #19](https://github.com/moreih29/nexus-core/issues/19), [GH #20](https://github.com/moreih29/nexus-core/issues/20)
**Plan session**: #6

---

## Summary

v0.10.0은 claude-nexus Plan session #7 업스트림 제안 2건(#19 Plan/Run quantitative guidelines, #20 memory operational policy + access tracking)의 **부분 수용** 결과를 실행한다. 수용 대상은 authoring layer 경계 안에 머무는 항목(artifact-coherence prose, conditional auto-pairing, exception catalog vocabulary, memory 3-category + access-log canonical schema, 5 behavioral principles docs). 거부 대상은 dispatcher·runtime·consumer-local 결정 영역(cap 수치, streaming 알고리즘, wave-time dedup, P1 구체 수치 등).

경계 판단의 canonical 근거로 `.nexus/context/boundaries.md §Canonical specifics의 증거 기준`을 신규 도입. 단일 consumer 증거로 올라온 구체 수치는 canonical로 박지 않는 원칙을 명시.

## Breaking Change Surface

### (1) `body_hash` 변경 — `skills/nx-plan` Step 7 재작성

body.md Step 7의 "Verification auto-pairing" 섹션이 conditional rule로 재구조화됨. `sha256:...` 변경. `verifyBodyHash`를 수행하는 consumer build는 기존 pin된 해시로 실패한다 → consumer 빌드 재실행 필수.

### (2) 신규 canonical vocabulary 2종 — manifest 구조 확장

- `vocabulary/task-exceptions.yml` (4 entries): `docs_only.coherent`, `docs_only.independent`, `same_file_bundle`, `generated_artifacts`.
- `vocabulary/memory_policy.yml` (5 sections): `categories` / `naming` / `access_tracking` / `forgetting` / `merge`.

`manifest.json`의 `vocabulary` 섹션에 `task_exceptions` 배열 + `memory_policy` 객체가 추가됨 (required). 기존 5 vocabulary entries(capabilities/categories/resume_tiers/tags/invocations)는 그대로.

### (3) 신규 canonical state schema — `memory-access.jsonl`

`conformance/state-schemas/memory-access.schema.json`이 추가됨. state schema count 4 → 5. `.nexus/state/{harness_id}/memory-access.jsonl`이 Shared filename convention에 등록됨 (agent-tracker.json 선례와 동일 패턴).

### (4) `vocabulary/invocations.yml` — memory_read_observation primitive 추가

4 → 5 entries. Consumer의 invocation map이 신규 primitive를 catch하지 못해도 기존 기능은 영향 없음(fallback_behavior 명시).

### (5) `vocabulary/tags.yml` — `[m]`/`[m:gc]` entries에 `prose_guidance` 필드 추가

schema `tagEntry`가 optional `prose_guidance` (minLength 40)를 허용하도록 확장. 기존 tags는 변경 없음.

## Consumer Action Checklist

### claude-nexus

1. `package.json`의 `@moreih29/nexus-core` devDependency를 `^0.10.0`으로 업데이트.
2. `bun run dev` (또는 consumer 빌드 명령)으로 재빌드. `verifyBodyHash` gate가 `skills/nx-plan` 새 body_hash를 수용.
3. **Conditional auto-pairing dispatcher 수정**:
   - 기존 코드: `owner=writer`이면 항상 reviewer pair 생성 → `owner=researcher`인 경우에도 tester/reviewer가 spawn되어 "artifact not found" 발생 (tester-artifact-gap.md 2026-04-10 incident 근거).
   - 신규 규칙: `engineer + runtime-behavior acceptance`만 tester pair. `writer + verifiable deliverable acceptance`만 reviewer pair. `researcher` · 순수 refactor · type-only · docs-adjacent (.md/frontmatter only) 제외.
   - docs-adjacent는 `vocabulary/task-exceptions.yml`의 `docs_only.coherent` / `docs_only.independent` entry로 분류.
4. **Dedup Layer 1 적용**: plan Step 7 task 생성 단계에서 draft task list를 스캔, `target_files` 겹침 시 `same_file_bundle` exception으로 merge. Layer 2 (wave-time intersection)는 consumer-local dispatcher 결정 — canonical 아님.
5. **dispatcher 거부 항목은 consumer-local로 이관**:
   - cap 수치 (예: 5 in-flight subagents) — 자체 설정값으로 유지.
   - pair-wise streaming spawn 알고리즘 — dispatcher 구현 재량.
   - Layer 2 dedup — 자체 wave scheduler에서 구현 또는 생략.
   - wave_id TUI grouping — UI presentation layer에서 자체 결정.
   - escalation wave pause/resume 상태 머신 — SubagentStop 핸들링 재량.
   - `tool-log.jsonl` wave_complete recording + cap recalibration — 내부 telemetry, canonical 아님.
6. **memory-access observation hook (선택적)**:
   - 구현 시 `.nexus/state/claude-nexus/memory-access.jsonl`에 4 필드 기록 (`path`, `last_accessed_ts`, `access_count`, `last_agent`).
   - `PostToolUse` hook에서 Read 이벤트 catch → `.nexus/memory/` 하위 경로만 필터 → upsert by path.
   - P1 자동 삭제 임계값 (예: 일수, cycle 수, access_count=0)은 프로젝트 cadence에 맞춰 consumer-local 설정. 3-signal intersection 구조는 준수.
   - git-backed deletion: `[m:gc]` 또는 P1 삭제 시 `git rm` + commit. commit message 포맷은 consumer 재량.
7. **exception catalog dispatcher 인식**:
   - `docs_only.coherent`: 파일 수·라인 수 임계값 waive, 1 writer + 1 reviewer pair.
   - `docs_only.independent`: N parallel writer tasks, each + 1 reviewer.
   - `same_file_bundle`: 같은 파일 targeting sub-tasks는 1 owner에 structured prompt로 merge.
   - `generated_artifacts`: build output 경로는 task count·file count에서 제외.

### opencode-nexus

1. `package.json`의 `@moreih29/nexus-core` devDependency를 `^0.10.0`으로 업데이트.
2. 빌드 재실행.
3. 동일 conditional auto-pairing 규칙을 opencode harness의 task 분해 로직에 반영 — `vocabulary/task-exceptions.yml`의 4 entries를 인식.
4. memory-access observation hook 구현 여부는 프로젝트 필요에 따라 판단. opencode-nexus의 observation primitive (Claude Code의 PostToolUse 대응)를 사용하여 `.nexus/state/opencode-nexus/memory-access.jsonl` 기록.
5. 거부 항목(cap, streaming, Layer 2, wave_id, escalation, tool-log) 모두 harness-local dispatcher 결정 — 필요에 따라 독립 구현.

## Rejected Items Reference (consumer-local decisions)

이 항목들은 nexus-core canonical에 포함되지 않는다. 각 consumer harness가 자기 프로젝트 cadence·context window·운영 특성에 맞춰 local 정책으로 설정한다.

**GH #19에서 거부**:
- `cap=5` hard 정수 (in-flight subagents upper bound) — Claude Code documented "3–5 teammates" 가이드에서 유도된 값. consumer concurrency profile에 따라 다름.
- pair-wise streaming spawn 알고리즘 (`DO` 완료 시 `CHECK` 즉시 spawn) — dispatcher scheduler 로직.
- Dedup Layer 2 (wave-time intersection check 직전 defer) — runtime dispatcher 결정.
- TUI `wave_id` grouping axis — UI presentation.
- Escalation serialization의 wave pause/resume 상태 머신 — SubagentStop 핸들링 구현.
- `tool-log.jsonl` wave_complete recording + cap recalibration 프로토콜 — harness-local telemetry.
- `run_parallel_dispatch` harness_keys entry (CA-7 조건부) — premature 도입.

**GH #20에서 거부**:
- `primer-` 범주 as memory category — `.nexus/context/`와 중복 역할 (저자 §7.3 자인). primer 성격 문서는 context/에 속함.
- `max_count: 1 primer` 제약 — multi-team 배포 edge case (저자 §8(b) 자인).
- 구체 수치: size limit 80/200 lines, gc trigger 15 files/60KB/20 cycles, P1 자동 삭제 180 days/6 cycles/access=0 — 모두 consumer cycle cadence 의존.
- P1 자동 삭제의 수치 enforcement (3-signal intersection 구조만 canonical, 수치는 consumer).
- git commit message 구체 포맷 — ops/supervision 영역.
- Resume 세션 미증분 규칙 — 하네스 resume 메커니즘 의존.
- 4-prefix enumeration regex의 강제 enforcement — 단일 consumer empirical. structural naming contract(lowercase kebab-case `.md`)는 canonical, prefix enumeration은 권고 수준.
