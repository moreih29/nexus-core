# Migration Guide: v0.8.x ŌåÆ v0.9.0

**Release date**: Unreleased (planned 2026-04-15)
**Type**: Minor breaking release (task_close scope reduction + rule enforcement)
**Affected consumers**: claude-nexus, opencode-nexus
**Tracking issues**: [GH #17](https://github.com/moreih29/nexus-core/issues/17), [GH #18](https://github.com/moreih29/nexus-core/issues/18)

---

## Summary

v0.9.0 reduces the scope of `task_close` to the nexus-core owned state files only, and removes harness-specific fields from the tool's return shape. Two breaking changes land together:

1. **Return field removed** ŌĆö `memoryHint.hadLoopDetection` is removed from the `task_close` return shape. This field reflected a claude-nexus-specific loop detection aggregation that does not generalize across harnesses.
2. **Delete side effects narrowed** ŌĆö `task_close` no longer deletes `edit-tracker.json` or `reopen-tracker.json`. These files are harness-local; their lifecycle is the responsibility of consumer harness session hooks. `task_close` now deletes only `plan.json` and `tasks.json` (both nexus-core owned).
3. **nexus-core contract for both tracker files removed** ŌĆö `edit-tracker.json` and `reopen-tracker.json` are no longer referenced as nexus-coreŌĆōmanaged outputs in `docs/nexus-outputs-contract.md`. They are harness-local state, to be placed under `.nexus/state/{harness-id}/` at consumer discretion.

A companion patch (Issue #17) corrects the `agent-tracker.json` narrative in `docs/nexus-state-overview.md` to reflect the array-of-entries structure already used in practice. That patch is non-breaking and requires no consumer action.

## Breaking Changes

### (a) `task_close` return shape ŌĆö `memoryHint.hadLoopDetection` removed

**Before (v0.8.x)**

```json
{
  "closed": true,
  "cycle": "2026-04-15T09:00:00.000Z",
  "branch": "main",
  "archived": { "plan": true, "decisions": 3, "tasks": 8 },
  "deleted": ["plan.json", "tasks.json"],
  "total_cycles": 5,
  "memoryHint": {
    "taskCount": 8,
    "decisionCount": 3,
    "cycleTopics": ["v0.9.0 scope reduction"],
    "hadLoopDetection": true
  }
}
```

**After (v0.9.0)**

```json
{
  "closed": true,
  "cycle": "2026-04-15T09:00:00.000Z",
  "branch": "main",
  "archived": { "plan": true, "decisions": 3, "tasks": 8 },
  "deleted": ["plan.json", "tasks.json"],
  "total_cycles": 5,
  "memoryHint": {
    "taskCount": 8,
    "decisionCount": 3,
    "cycleTopics": ["v0.9.0 scope reduction"]
  }
}
```

`hadLoopDetection` is absent from the return object. Any consumer code that reads `result.memoryHint.hadLoopDetection` will receive `undefined`.

---

### (b) `task_close` delete side effects ŌĆö `edit-tracker.json` and `reopen-tracker.json` removed

**Before (v0.8.x)**: `task_close` deleted `plan.json`, `tasks.json`, `edit-tracker.json`, and `reopen-tracker.json` (all within `.nexus/state/`).

**After (v0.9.0)**: `task_close` deletes only `plan.json` and `tasks.json`. The `deleted` array in the return value will contain only these two entries when they exist. `edit-tracker.json` and `reopen-tracker.json` are no longer touched by `task_close` under any circumstance.

```
# Before (v0.8.x) ŌĆö deleted array could contain:
["plan.json", "tasks.json", "edit-tracker.json", "reopen-tracker.json"]

# After (v0.9.0) ŌĆö deleted array contains only:
["plan.json", "tasks.json"]
```

---

### (c) nexus-core contract for `edit-tracker.json` and `reopen-tracker.json` removed

Both tracker files are no longer governed by nexus-core normative contracts (`docs/nexus-outputs-contract.md`). Any consumer reference to them as nexus-coreŌĆōdefined outputs is invalid from v0.9.0 onward. These files are harness-local state; consumer harnesses manage them entirely within their own session hook lifecycle.

Recommended harness-local paths (consumer decision, not nexus-core prescription):

```
.nexus/state/{harness-id}/edit-tracker.json
.nexus/state/{harness-id}/reopen-tracker.json
```

## Consumer Action Required

### claude-nexus

1. **Remove `hadLoopDetection` references**: find all code paths that read `result.memoryHint.hadLoopDetection` from a `task_close` return value and remove or replace them. The field is no longer returned.

2. **Move tracker cleanup to `session_end` hook**: `edit-tracker.json` and `reopen-tracker.json` are no longer deleted by `task_close`. Add explicit cleanup for these files in the `session_end` hook (or equivalent termination path). Recommended paths:

   ```
   .nexus/state/claude-nexus/edit-tracker.json
   .nexus/state/claude-nexus/reopen-tracker.json
   ```

   The cleanup logic should check for file existence before attempting deletion (the files may not be present in every session).

3. **Re-read `rule:neutral-tool-side-effect`** in `.nexus/rules/neutral-principles.md` ŌĆö the new rule codifies why harness-local file management must stay in consumer session hooks, not in nexus-core tool contracts.

### opencode-nexus

1. **Remove `hadLoopDetection` references**: find all code paths that read `result.memoryHint.hadLoopDetection` from a `task_close` return value and remove or replace them. The field is no longer returned.

2. **No additional tracker action required**: opencode-nexus does not implement `edit-tracker.json` or `reopen-tracker.json`, so the deletion side effect removal is a no-op for this harness.

## Consumer Action Checklist

Apply all applicable items before upgrading to v0.9.0.

- [ ] Search for `hadLoopDetection` in your codebase and remove or replace all references to this field.
- [ ] (claude-nexus only) Add `edit-tracker.json` cleanup to your `session_end` hook, targeting `.nexus/state/claude-nexus/edit-tracker.json`.
- [ ] (claude-nexus only) Add `reopen-tracker.json` cleanup to your `session_end` hook, targeting `.nexus/state/claude-nexus/reopen-tracker.json`.
- [ ] Update your internal documentation to reflect that these tracker files are harness-local and are not managed by nexus-core tool contracts.
- [ ] Verify that `task_close` invocations in your test suite do not assert `hadLoopDetection` presence or a specific `deleted` array length that includes the tracker files.

## Migration policy

`edit-tracker.json` and `reopen-tracker.json` are session-scoped files. No user-initiated data migration is required. Harnesses that previously relied on `task_close` to delete these files MUST add `session_end` cleanup. Any instance of these files left behind by a session running on v0.8.x will not be cleaned up by a subsequent v0.9.0 `task_close` call ŌĆö the `session_end` hook is the correct point to handle this.

No data migration tooling is provided by nexus-core for this change.

## Rationale

**Plan session #5 Issue #18** (2026-04-15): `task_close` scope reduction was decided under the principle that nexus-core MCP tool contracts must not reference or manage harness-local state. The key references:

- `rule:neutral-tool-side-effect` (`.nexus/rules/neutral-principles.md`, v0.9.0 ņŗĀņäż): formalizes that nexus-core MCP tool contracts may only declare side effects on `conformance/state-schemas/*.json`-registered files (`plan.json`, `tasks.json`, `history.json`, `agent-tracker.json`). `edit-tracker.json` and `reopen-tracker.json` are not registered in conformance schemas.
- `hadLoopDetection` was a claude-nexusŌĆōspecific behavioral flag surfaced through a shared tool return. Its presence in a harness-neutral contract was an architectural boundary violation.

**Plan session #5 Issue #17** (patch, no breaking change): `agent-tracker.json` narrative updated to reflect the array-of-entries structure already implemented. No consumer action required for this patch.
