# 에이전트 조율 모델

> nexus-core의 핵심 기여 — 3 하네스가 공유하는 조율 골격. Lead + 9 에이전트 + 권한 · 재개 정책.

## 1. 구성

### 1-1. Lead

사용자와의 단일 대화 채널. 작업 분배 · 하위 에이전트 결과 종합 · 의사결정 책임. 단순 분배자가 아닌 능동적 참여자 — 자기 의견을 형성하고 근거 있는 반론을 제기한다.

### 1-2. 3 카테고리 · 9 에이전트

| 카테고리 | 역할 | 멤버 |
|---|---|---|
| **HOW** | 분석 · 설계 · 판단 | architect · designer · postdoc · strategist |
| **DO** | 실행 · 구현 · 생성 | engineer · researcher · writer |
| **CHECK** | 검증 · 리뷰 · 테스트 | reviewer · tester |

## 2. 에이전트 권한

각 에이전트는 사용 가능한 도구 집합이 정해져 있다. 권한은 추상 단위로 표현되며 하네스가 자기 도구로 매핑한다.

카테고리별 권한 패턴:

| 카테고리 | 파일 쓰기 | 셸 실행 | 산출물 |
|---|---|---|---|
| HOW | 금지 | 금지 | 분석 권고 |
| DO | 허용 | engineer만 | 코드 · 문서 · 조사 결과 |
| CHECK | 금지 | tester만 | 검증 보고 |

Lead는 작업 할당 시 이 패턴을 참조한다 — 코드 작성 = engineer, 문서 작성 = writer, 외부 조사 = researcher 등.

## 3. 재개 정책

하위 에이전트 호출 시 새로 생성할지 재개할지 결정하는 정책.

| 등급 | 의미 |
|---|---|
| `persistent` | 같은 이슈 내 기본 재개. 이슈 간 재개는 Lead 명시 동의 시 |
| `bounded` | 같은 산출물만 재개. 재개 사이 타 에이전트 개입 시 재읽기 의무 |
| `ephemeral` | 항상 새로 생성 |
